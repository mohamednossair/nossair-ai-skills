# Git Skill

## References

## Guidelines
- Use feature/bugfix/docs/chore branches.
- Conventional commits; small, logical changesets.
- Rebase for clean history unless policy dictates merge.

## Checklist
- [ ] Clear PR description and scope
- [ ] Commits atomic; messages meaningful
- [ ] CI green; reviewers assigned
